
# Top P
`top_p` 参数是生成式语言模型中的一个重要采样参数，用于控制模型生成文本时的多样性和确定性。

### 参数含义
- **核采样（Nucleus Sampling）**：模型会计算所有可能的下一个 token 的概率，按概率从高到低排序，然后累积概率直到达到 `top_p` 值，只从这个范围内选择下一个 token。
- **取值范围**：0-1之间
  - 值越接近 0：模型选择范围越窄，生成内容越确定、保守
  - 值越接近 1：模型选择范围越宽，生成内容更多样、可能更有创意

### 主流开源模型推荐设置
| 模型系列 | 推荐 top_p 范围 | 说明 |
|---------|---------------|------|
| GPT 系列 | 0.7-0.9 | OpenAI 官方推荐 0.9 |
| Llama 系列 | 0.8-0.95 | Meta 官方推荐 0.9 |
| Mistral 系列 | 0.7-0.9 | 官方默认 0.8 |
| Gemma 系列 | 0.8-0.9 | Google 官方推荐 0.9 |
| Claude 系列 | 0.8-0.9 | Anthropic 推荐 0.9 |

### 任务适配建议
- **代码生成**：0.7-0.8（需要更高的确定性）
- **创意写作**：0.85-0.95（需要更多样性）
- **问答任务**：0.8-0.9（平衡确定性和多样性）
- **翻译任务**：0.75-0.85（需要准确表达）

### 与其他参数的配合
`top_p` 通常与 `temperature` 参数配合使用：
- 较高的 `temperature` 配合适中的 `top_p` 可以产生更多样的输出
- 较低的 `temperature` 配合适中的 `top_p` 可以产生更确定的输出

#

以下是主流开源模型中各参数的推荐设置，基于当前代码中的参数列表：


### 1. **模型架构参数**
| 参数 | 说明 | 主流模型推荐值 | 备注 |
|------|------|----------------|------|
| `hidden_size` | 隐藏层维度 | 因模型而异：<br>- Llama 2 7B: 4096<br>- Mistral 7B: 4096<br>- Gemma 7B: 3072 | 此参数为模型结构固定值，不同模型架构有不同设置 |
| `num_hidden_layers` | 隐藏层数量 | 因模型而异：<br>- Llama 2 7B: 32<br>- Mistral 7B: 32<br>- Gemma 7B: 28 | 同样为模型结构固定值，层数越多模型能力通常越强 |
| `use_moe` | 是否使用MoE架构 | 视模型而定：<br>- Mixtral 8x7B: 1<br>- 多数模型: 0 | MoE架构主要用于大型模型（如Mixtral），可在参数量相近时提升性能 |
| `inference_rope_scaling` | RoPE位置编码外推 | 长文本任务: `True`<br>短文本任务: `False` | 启用后可处理更长的上下文（如4倍长度），主流模型如Llama 2支持 |


### 2. **生成控制参数**
| 参数 | 说明 | 主流模型推荐值 | 备注 |
|------|------|----------------|------|
| `max_new_tokens` | 最大生成长度 | 2048-4096 | 平衡生成质量和速度，具体根据任务需求调整 |
| `temperature` | 生成温度 | 0.7-0.9 | 0.7-0.8适合需要准确性的任务（如问答），0.8-0.9适合需要创意的任务 |
| `top_p` | nucleus采样阈值 | 0.8-0.95 | 0.8-0.9平衡多样性和准确性，0.9-0.95增加创意性 |
| `repetition_penalty` | 重复惩罚 | 1.0-1.2 | 1.05-1.1适合一般场景，1.1-1.2适合需要更严格避免重复的场景 |


### 3. **其他参数**
| 参数 | 说明 | 主流模型推荐值 | 备注 |
|------|------|----------------|------|
| `historys` | 携带历史对话轮数 | 2-4（偶数） | 对话场景下保留适当历史可提升连贯性，过多可能导致上下文过长 |
| `device` | 运行设备 | `cuda`（如有GPU） | 主流模型优先使用GPU加速，无GPU时使用`cpu` |


### 主流模型具体参考配置
| 模型 | temperature | top_p | repetition_penalty | max_new_tokens |
|------|-------------|-------|-------------------|----------------|
| Llama 2 | 0.7-0.9 | 0.9 | 1.0-1.1 | 2048-4096 |
| Mistral | 0.7-0.85 | 0.8-0.9 | 1.0-1.1 | 2048-4096 |
| Gemma | 0.8 | 0.9 | 1.05 | 2048 |
| Claude | 0.8 | 0.9 | 1.05 | 4096 |


### 调整建议
- **任务导向调整**：
  - 代码生成/数学推理：降低temperature（0.6-0.7），提高top_p（0.9-0.95）
  - 创意写作/故事生成：提高temperature（0.8-0.9），提高top_p（0.9-0.95）
  - 问答/信息提取：适中temperature（0.7-0.8），适中top_p（0.85-0.9）
- **长文本处理**：启用`inference_rope_scaling`，调整`max_new_tokens`至4096或更高
- **避免重复**：如遇生成内容重复，增加`repetition_penalty`至1.1-1.2

当前代码中的参数设置（如temperature=0.85、top_p=0.85、repetition_penalty=1.05）基本符合主流模型的推荐范围，适合一般对话场景。
